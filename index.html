Module 1: Introduction to Web Development
Objective: In this module, you'll get acquainted with the fundamentals of web development. We'll start by understanding the core distinctions between frontend and backend development. Additionally, we'll introduce you to the essential tools and environments that will be pivotal in your journey.

Topic 1.1: Overview of Web Development

What is Web Development?

Web development refers to the process of creating websites and web applications. It involves a combination of various technologies, programming languages, and tools to bring a concept or idea to life on the internet.

Technologies in Web Development:

Frontend Technologies:

HTML (HyperText Markup Language):

HTML is the standard markup language for creating web pages. It provides the basic structure and content of a webpage.

CSS (Cascading Style Sheets):

CSS is used to style and format the layout of a webpage. It controls the visual presentation of the HTML elements.

JavaScript:

JavaScript is a programming language that adds interactivity and dynamic behavior to web pages. It's essential for creating interactive elements and controlling the DOM (Document Object Model).

Backend Technologies:

Server-side Languages:

Languages like JavaScript (Node.js), Python, Ruby, Java, and PHP are commonly used for backend development.

Databases:

Databases store and manage the data of a website. Popular databases include MySQL, PostgreSQL, MongoDB, and SQLite.

Frameworks and Libraries:

Backend frameworks like Express.js (for Node.js), Django (for Python), Ruby on Rails (for Ruby), etc., provide pre-built structures and libraries for faster development.

Web Development Process:

Planning and Design:

Define project goals, target audience, and create wireframes/mockups for the website's layout and functionality.

Frontend Development:

Write HTML, CSS, and JavaScript code to create the user interface and interactivity of the website.

Backend Development:

Implement server-side logic, connect to databases, and handle requests and responses.

Testing:

Ensure the website functions as intended, is compatible across different browsers, and is free of bugs or errors.

Deployment:

Make the website accessible on the internet by hosting it on a server or a cloud platform.

Maintenance and Updates:

Regularly update content, fix bugs, and implement new features or improvements as needed.

Topic 1.2: Key Components of Web Development

Frontend Development:

Frontend development focuses on the user interface and user experience of a website or web application. It involves creating the visual elements (What you can see on the computer screen) that users interact with in their web browsers.

It involves writing code (HTML, CSS, JavaScript) that controls the visual presentation and behavior of a site in the user's browser.

Backend Development:

Backend development focuses on the server-side logic and data management of a web application. It involves handling requests from the frontend, interacting with databases, and performing tasks on the server. It manages the application functionality.

It involves writing code (server-side languages like JavaScript, Python, Ruby, etc.) that handles requests, processes data, and interacts with databases.

Full Stack Development:

Full-stack development involves working on both the frontend and backend of a web application. A full-stack developer has proficiency in both client-side and server-side technologies.

Full stack developers have expertise in both frontend and backend development. They are capable of handling the entire web development process.

Topic 2.1: HTML - Building the Structure

HTML stands for (HyperText Markup Language). It's the standard language used to create web pages. Let's break it down in a beginner-friendly way:

What is HTML?

HTML is the foundation of every webpage you see on the internet. It's like the skeleton that gives structure to a page. Imagine a webpage as a house - HTML forms the framework, while CSS (Cascading Style Sheets) is like the paint and decoration.

How Does HTML Work?

HTML uses a system of "tags". These tags are like labels that tell your web browser how to display the content. Each tag has an opening <tag> and a closing </tag>. For example, <h1> is a tag for a main heading, and </h1> marks the end of that heading.

Basic Structure of an HTML Page:

                
<!DOCTYPE html>
<html>
<head>
    <title>Page Title</title>
</head>
<body>
    <h1>This is a Heading</h1>
    <p>This is a paragraph.</p>
</body>
</html>
index.html

Output:

How to Write HTML?

You can use a simple text editor on your computer or a code editor app on your phone. Just type out the HTML code and save it with a ".html" extension.

Example:

  • Open a text editor on your computer or a code editor app on your phone.
  • Type or copy the code above.
  • Save the file with a ".html" extension (e.g., "my_first_page.html").
  • Open it with a web browser. You'll see "This is a Heading" as a big heading, and "This is a paragraph." as a regular paragraph.

Exercise 1: Creating a Simple Web Page

Objective: Build a basic web page with headings, paragraphs, and a link.

Steps:

Open a Text Editor:

  • On a computer, you can use Notepad (Windows), TextEdit (Mac), or a code editor like Visual Studio Code, Sublime Text, etc.
  • On a phone, you can use apps like "QuickEdit Text Editor" (Android) or "Textastic Code Editor" (iOS).

Start with HTML Structure:

  • Type "<!DOCTYPE html>" at the very beginning. This declares the document as an HTML5 document.
  • Then, add <html>, <head>, and <body> tags.

Inside the <html> section, add <title> tag to give your page a title (this will be displayed on the browser's title bar).

Inside the <body> section, add:

  • <h1> for a main heading.
  • <p> for a paragraph of text.
  • <p> to create a link. Use the href attribute to specify the URL.

Save the file with an .html extension (e.g., my_first_page.html).

Open it with a web browser (double-click the file). You'll see your web page!

Exercise 2: Adding an Image

Objective: Add an image to your web page.

Steps:

In your HTML file, below the paragraph, add an <img> tag.

  • Use the "src" attribute to specify the path to the image file.
  • Use the "alt" attribute to provide alternative text (important for accessibility).

Save the file and refresh your browser. You should now see the image on your page.

Elements, Tags, and Attributes

Understanding HTML elements, tags, and attributes is fundamental to creating web pages. Let's break down these concepts:

HTML Elements:

An HTML element is a fundamental building block of a web page. It consists of a start tag, content, and an end tag. The start tag and end tag together define the structure of the element. For example: 

                
<p>This is a paragraph.</p>
index.html

In this example:

  • "<p>" is the start tag.
  • "</p>" is the end tag.
  • "This is a paragraph" is the content of the element.

HTML Tags:

HTML tags are keywords that define how elements are displayed on a web page. They consist of an opening tag (start tag) and a closing tag (end tag), which wrap around the content. Tags are enclosed in angle brackets "< >".

There are two types of tags:

  • Opening Tags ("<tag>"): They mark the beginning of an element.
  • Closing Tags ("</tag>"): They mark the end of an element.

In some cases, certain elements are self-closing and don't have a closing tag. For example, the line break element: 

                
<br>
index.html

HTML Attributes:

Attributes provide additional information about an element and are always included in the opening tag. They are made up of a name and a value, separated by an equals sign ("=") and enclosed in double or single quotes. For example: 

                
<a href="https://www.example.com">Click here<a>
index.html

In this example:

  • "href" is the attribute name.
  • "https://www.example.com" is the attribute value.

Attributes can be used to specify things like links, image sources, input types in forms, and more.

Nesting Elements:

HTML elements can be nested within each other, meaning one element can contain another. For example: 

                
<div>
  <h1>This is a heading</h1>
  <p>This is a paragraph</p>
</div>
index.html

Output:

Here, the "div" element contains an "h1" element and a "p" element.

Closing Thoughts:

  • Elements, tags, and attributes work together to create the structure and content of a web page.
  • Properly nested and balanced tags are crucial for well-formed HTML.
  • Attributes provide additional information and functionality to elements.

N.B: HTML forms the foundation of web development. Mastering these basic concepts will enable you to create more complex and interactive web pages.

Attributes and Their Usage

  1. "id" Attribute:

    2. Usage: Provides a unique identifier for an HTML element.

    Example: 

                      
<div id="header">
  <!-- Content of the header -->
</div>
    index.html
  2. "class" Attribute:

    3. Usage: Assigns one or more class names to an element. Useful for applying CSS styles or grouping elements.

    Example: 

                      
<p class="important-text">
  This is an important paragraph.
</p>
    index.html
  3. "src" Attribute (for "<img>" and "<script>" elements):

    4. Usage: Specifies the URL the link points to.

    Example: 

                      
<img src="image.jpg" alt="Description of the image">
    index.html
  4. "href" Attribute (for "<a>") elements:

    5. Usage: Specifies the URL the link points to.

    Example: 

                      
<a href="https://www.example.com">Click here to visit Example.com</a>
    index.html
  5. "alt" Attribute (for "<img>") elements:

    6. Usage: Provides alternative text for an image, which is displayed if the image fails to load.

    Example: 

                      
<img src="image.jpg" alt="Description of the image">
    index.html
  6. "title" Attribute:

    7. Usage: Adds a title or tooltip to an element.

    Example:

    Example:

    11. Usage: Provides a hint or example of the expected value for an input field.

    Example:

    index.html
